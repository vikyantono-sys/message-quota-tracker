One-time setup required before live refresh works. Until this is done, the page shows the embedded snapshot below.
  1. In the qraved-id Google Cloud project, create an OAuth 2.0 Client ID (type: Web application). Under Authorized JavaScript origins, add the GitHub Pages origin this file is hosted at (e.g. https://yourorg.github.io).
  2. Open this file's source and set CONFIG.CLIENT_ID, CONFIG.PROJECT_ID and CONFIG.LOCATION at the top of the script.
  3. Grant each AM/Ops Google account BigQuery Data Viewer + BigQuery Job User on the project (ideally scoped to a read-only view in agent_safe_views).
  4. Quotas are read live from the internal_gsheet.Quota_Tracker Google Sheet, so each user also needs view access to that sheet — the page requests Google Drive read scope to load it. (To avoid the Drive dependency, you can instead materialize the sheet into a native BigQuery table and point the query at it.)
  5. Commit and push to GitHub Pages, then share the page URL.

Live data is pulled straight from BigQuery in your browser when you click Refresh from BigQuery above — you'll sign in with your Google account the first time. The query run is shown below for transparency.  ·

    How periods are measured  · 
    Quotas are read live from the internal_gsheet.Quota_Tracker sheet — edit them there and Refresh.
    Monthly quotas count within the calendar month of the “as of” date (resets each month).
    3 Months quotas use a rolling 3-month bucket anchored on the live date.
    1 Time quotas are cumulative from the merchant's live date (a one-time allocation, no reset); a 1-Time row with no live date shows as “needs attention” until the date is filled in the sheet. A 1-Time merchant who has sent more than its allocation will correctly read over quota.
    Count-based quotas track SUM(message_sent); value-based quotas track SUM(message_charge_total) (switchable to merchant-borne above).
    Calendar-month is assumed for monthly resets. A merchant in the sheet whose name has no recognized source_table mapping is flagged — add it to the map (or add a source_table column to the sheet).
